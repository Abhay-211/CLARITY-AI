Analysis & Insights

After cleaning, new features (`slang_length` and `meaning_length`) were created to analyze the dataset's characteristics. The report included a boxplot, histogram, and scatterplot to find insights.

Distribution of Slang Length

The analysis found the `slang_length` is heavily **right-skewed (skewness: 1.56)**. This is confirmed by the chart, showing that most slang terms are very short (3-4 characters), with a few long outliers.

Slang Length vs. Meaning Length

The report found a **moderate positive correlation (r = 0.476)** between the length of the slang term and the length of its meaning. The chart visualizes this: while there's a lot of variance, longer slang terms *tend* to have longer meanings.

About the Process

This interactive application is a direct translation of the process outlined in the source "Social Media Slang" Jupyter Notebook. The notebook details a complete data science workflow, which has been adapted for this web interface.

1. Data Ingestion: The process begins by downloading the "Social Media Abbreviations" dataset from Kaggle.

2. Cleaning & Processing: The raw CSV data is loaded into a pandas DataFrame and undergoes significant cleaning (as detailed in the 'Data Exploration' tab) to make it reliable.

3. "Model" Building: The "model" in this report is a simple but effective Python dictionary (or in this app's case, a JavaScript object). The cleaned `slang` column is used as the key, and the `meaning` column as the value. This creates a high-speed, 1-to-1 lookup table with 1,446 entries.

4. "Prediction" Function: The `normalize_slang` function is the "prediction" engine. It takes a user's sentence, splits it into words, and for each word, it:

  • Converts the word to lowercase.
  • Strips any punctuation (like `lol!` becoming `lol`).
  • Looks up the cleaned word in the "model" dictionary.
  • If found, it replaces the word with its meaning. If not, it keeps the original word.

5. Deployment: The report's final step was an interactive console app. This SPA replicates and enhances that final step in the 'Normalizer' tab, using JavaScript to run the exact same logic in your browser.
